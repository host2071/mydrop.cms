# MyDrop CMS

[MyDrop.io](https://mydrop.io/?pid=52817) сервис поиска и предварительной оценки: сервис помогает находить освобождающиеся домены до того, как они снова попадут в свободную регистрацию. Для покупателя это закрывает главную боль: не нужно вручную проверять сотни имен, следить за датами удаления и гадать, есть ли у домена история, трафик или ценность для SEO. В MyDrop.io можно отобрать освобождающиеся домены ru, com и еще 100 доменных зон по тематике, показателям, возрасту, ссылочному профилю и другим параметрам, а затем выбрать кандидатов для покупки. Сама покупка и подача заявки выполняются через [expired.ru](https://expired.ru/?p=187219), где пользователь может купить освобождающийся домен через сервис регистрации и аукционов. Такой сценарий экономит время: MyDrop.io помогает быстро найти сильные варианты, а expired.ru закрывает этап оформления заявки и перехвата. Это особенно полезно SEO-специалистам, вебмастерам, инвесторам и предпринимателям, которым важны домены с историей, понятными метриками и шансом на успешную регистрацию.

Эта версия CMS доработана относительно стандартной: в нее добавлена возможность формировать CSV-экспорт статей и данных страниц. Такой файл можно использовать для дальнейшего импорта материалов в WordPress или любую другую систему управления сайтами, которая поддерживает загрузку контента из CSV.

## Что есть в проекте

- `mydrop.cms.utf8.php` - основная версия CMS с редактором, массовой заменой, удалением элементов по CSS-селектору и экспортом CSV.
- `mydrop.utf8.php` - более ранний/упрощенный вариант CMS.
- `didom/` - локально подключенная библиотека DiDom для разбора HTML и работы с CSS-селекторами.

## Требования

- PHP 7.4+ или 8.x.
- Расширение `pdo_sqlite`.
- Доступ PHP к записи в корневую папку проекта, чтобы CMS могла создать `.mdcms_data`.
- Интернет-доступ в браузере для загрузки CDN-зависимостей интерфейса: Bootstrap, jQuery, jsTree, CodeMirror и TinyMCE.

## Подготовка данных

CMS ожидает CSV-файл с именем домена из константы `MDCMS_DOMAIN`. В текущей конфигурации это:

```text
viabit.ru.csv
```

Каждая строка CSV должна сопоставлять URL и HTML-файл в формате, который использует MyDrop:

```text
viabit.ru/path;viabit.ru/ab/cd/abcdef0123456789abcdef0123456789.html
```

HTML-файлы должны лежать в корне проекта в подпапках по первым символам хеша:

```text
ab/cd/abcdef0123456789abcdef0123456789.html
```

При первом запуске CMS создаст папку `.mdcms_data`, базу SQLite `structure.db` и заполнит структуру страниц из CSV.

## Настройка

Основные параметры находятся в начале `mydrop.cms.utf8.php`:

```php
const MDCMS_VERSION = '1.1';
const MDCMS_PASSWORD = '123qwe';
const MDCMS_DOMAIN = 'viabit.ru';
```

Перед публикацией обязательно смените `MDCMS_PASSWORD`. Если пароль оставить пустым, проверка доступа будет отключена.

## Запуск

Для локальной проверки можно использовать встроенный PHP-сервер:

```bash
php -S 127.0.0.1:8080
```

Затем откройте в браузере:

```text
http://127.0.0.1:8080/mydrop.cms.utf8.php
```

На хостинге достаточно загрузить файлы проекта, CSV и HTML-дамп в одну директорию и открыть `mydrop.cms.utf8.php`.

## Основные возможности

- Редактирование страниц через WYSIWYG-редактор TinyMCE.
- Редактирование исходного HTML через CodeMirror.
- Массовая текстовая замена по всем HTML-файлам.
- Удаление блоков по CSS-селектору, например `.social_share.clearfix`.
- Настраиваемый экспорт CSV файл по CSS-селекторам и атрибутам, например `meta[name="description"]@content`.

## Служебные файлы

Папка `.mdcms_data` создается автоматически и не хранится в Git. В ней находятся база структуры, правила экспорта и временные экспортированные CSV-файлы.
