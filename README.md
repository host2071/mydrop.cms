# MyDrop CMS

Небольшая PHP-панель для редактирования статических HTML-файлов домена, выгруженного из MyDrop.io. Проект рассчитан на размещение рядом с CSV-картой сайта и HTML-файлами, после чего позволяет редактировать страницы, выполнять массовую замену и экспортировать данные в CSV.

## Что есть в проекте

- `mydrop.cms.utf8.php` - основная версия CMS с редактором, массовой заменой, удалением элементов по CSS-селектору и экспортом CSV.
- `mydrop.utf8.php` - более ранний/упрощенный вариант CMS.
- `didom/` - локально подключенная библиотека DiDom для разбора HTML и работы с CSS-селекторами.
- `nbproject/` - настройки PHP-проекта для NetBeans.

## Требования

- PHP 7.4+ или 8.x.
- Расширение `pdo_sqlite`.
- Доступ PHP к записи в корневую папку проекта, чтобы CMS могла создать `.mdcms_data`.
- Интернет-доступ в браузере для загрузки CDN-зависимостей интерфейса: Bootstrap, jQuery, jsTree, CodeMirror и TinyMCE.

## Подготовка данных

CMS ожидает CSV-файл с именем домена из константы `MDCMS_DOMAIN`. В текущей конфигурации это:

```text
industrial-craft2.ru.csv
```

Каждая строка CSV должна сопоставлять URL и HTML-файл в формате, который использует MyDrop:

```text
industrial-craft2.ru/path;industrial-craft2.ru/ab/cd/abcdef0123456789abcdef0123456789.html
```

HTML-файлы должны лежать в корне проекта в подпапках по первым символам хеша:

```text
ab/cd/abcdef0123456789abcdef0123456789.html
```

При первом запуске CMS создаст папку `.mdcms_data`, базу SQLite `structure.db` и заполнит структуру страниц из CSV.

## Настройка

Основные параметры находятся в начале `mydrop.cms.utf8.php`:

```php
const MDCMS_VERSION = '1.1';
const MDCMS_PASSWORD = '123qwe!@#';
const MDCMS_DOMAIN = 'industrial-craft2.ru';
```

Перед публикацией обязательно смените `MDCMS_PASSWORD`. Если пароль оставить пустым, проверка доступа будет отключена.

## Запуск

Для локальной проверки можно использовать встроенный PHP-сервер:

```bash
php -S 127.0.0.1:8080
```

Затем откройте в браузере:

```text
http://127.0.0.1:8080/mydrop.cms.utf8.php
```

На хостинге достаточно загрузить файлы проекта, CSV и HTML-дамп в одну директорию и открыть `mydrop.cms.utf8.php` через веб-сервер.

## Основные возможности

- Редактирование страниц через WYSIWYG-редактор TinyMCE.
- Редактирование исходного HTML через CodeMirror.
- Массовая текстовая замена по всем HTML-файлам.
- Удаление блоков по CSS-селектору, например `.social_share.clearfix`.
- Настраиваемый экспорт CSV по CSS-селекторам и атрибутам, например `meta[name="description"]@content`.

## Служебные файлы

Папка `.mdcms_data` создается автоматически и не хранится в Git. В ней находятся база структуры, правила экспорта и временные экспортированные CSV-файлы.
